18)  =======> LifeCycle Events

Hooks is also Lifecycle Events 
Special methods is also Lifecycle Events
LifeCycle Method is alo LifeCycle Events

a) Special methods/hooks provided by NestJS.
b) Automatically called at different stages of a module/service/component's life.
c) Used to perform actions during creation or destruction.

            Why are they useful?
a) Helps run some code when app/module/service is initialized.
b) Helps run cleanup code  when  app/module/service is destroyed.
c) Useful for tasks like DB connections, logging, resource cleanup, etc.,

            When are they triggered?
a) onModuleInit()            --> Called when the module is initialized.
b) onModuleDestroy()         --> Called before the module is destroyed.
c) afterModuleInit()         --> Called after all modules are initialized (optional).
d) onApplication Bootstrap() --> When app is fully bootstrapped.
e) onApplicationShutdown()   --> Called when app is shuting down.

29) Data Relationship in MongoDB 
a) Connection between documents in collection.
b)For make Relation ship in mongoDb we have two technique
        1) Embedding : Storing related data inside the same document.
        2) Refrencing : Storing related data in separate documents and linking by IDs.

        RelationShip Types
        1) One-to-One Relationship
        2) One-to-Many Relationship
        3) Many-to-Many Relationship

    Pros of Embedding
        Fast data retrieval (Single document).
        Simple for small related data.

    Cons of Embedding
        Document size grows big.
        Hard to update depply nested data.

    Pros of Refrencing
        Keeps documents small.
        Easier to update and manage large data.
    Cons of Refrencing
        Requires extra queries or joins (populate() in Mongoose).
        Slightly slower reads.
