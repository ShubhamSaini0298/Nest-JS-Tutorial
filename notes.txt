{#} DTO (Data Transfer Object)

a) An object that carries data between layers (like from client to backend).
b) Used to define the shape of incoming request data.
c) Ensures only required data is passed (security + validation).

{#} Interface
a) Interfaces define the structure (type) of an object.
b) Help write clean, structured, type-safe code.
c) Used for both request(DTOs) and response objects.

13)=========>>>>>>>>> Custom Pipes

a) Pipes are used to transform or validate incoming data.
b) Nest js allows you to create your own Custom Pipes.
c) They can be used for custom validation, data transformation, or business logic filtering.
        ------important-----
d) A Pipe runs before the data hits the route handler (controller method).
e) You can apply pipes at method level, controller level, or globally.
f) Custom Pipes implement the Pipe Transform interface.

==> In myname folder we implement the custom pipe implementation and in common/pipes/uppercase  have the method of custom pipe that make the  name in uppercase.

14) =======>>>>>>>How to Protect Routes using Guards 

a) It means restricting  access to specific  API routes.
b) Only authorized users (like logged-in users or admins) can access them.

        ======Guards
c) Guards are classes that implement logic to decide whether a request is allowed or not.
d) Thay implement the CanActivate interface and run before the route handler.
e) Mostly used for authentication or  authorization.
        ===== Why we used Guards
f) Top secure private routes.
g) To avoid duplicating checks in every controller.
h) To build role-based access control systems.
